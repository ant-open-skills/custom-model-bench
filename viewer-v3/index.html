Thesis

v2 gave you a sortable leaderboard. v3 answers a sharper question: "Which model should I ship for my product?" Not "which benchmark winner?" — Opus tops math benchmarks and still makes for a bad, expensive support agent. The Frontier chart shows only the models that aren't dominated on your personal trade-off; everything else is hidden as noise.

Three screens

  • Frontier — cost × latency × reliability. Move the persona slider, watch non-dominated models survive. The answer to "which?"
  • Trace diff — same prompt × N candidates in a stacked column view, tool calls inlined. The answer to "why?"
  • Behavior — per-candidate turns / tokens / tool mix. Surfaces the Sonnet-Agent-SDK vs Sonnet-AI-SDK verbosity gap (9 turns, 3234 tokens vs 4 turns, 772 tokens). Same model, different harness — wildly different runs.

Quality axis — honest

Today "quality" = JSON-validity / success-rate. Labeled as such everywhere. A quality_rubric slot is reserved in the fit weights; when the backend ships rubric grading, the frontier chart upgrades without redesign.

What this replaces

If v3 earns its keep, it replaces v2 entirely. If not, v2 is untouched and we revert cleanly.
